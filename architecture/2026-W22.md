# Architecture Memo – 2026-W22

## Theme of the Week
Establishment of `agentic-practices` as the independent meta/coordination layer for all agent work across multiple domain repos.

## Observed Patterns
- Domain repos (`execute-ai`, `personal-workflows`) benefit from staying focused on their specific work without being polluted by agent coordination logic.
- Agent coordination (skills, chronicles, future agents) needs its own home so it can remain objective and reusable.

## Design Proposals
- **Proposal**: Keep all future agents (Architect, Closer, Researcher) under `agentic-practices/skills/`
  - **Problem**: Previously the Chronicler lived inside `execute-ai`, creating coupling.
  - **Recommendation**: Treat `agentic-practices` as the single source of truth for agent definitions and outputs.
  - **Rationale**: Enables clean separation and makes the agent layer portable across different client engagements.

## Cross-Repo Observations
- `personal-workflows` is moving fast on finance automation tooling.
- `execute-ai` / `moura-client-portal` is the current production solution.

## Recommended Experiments
- Continue with weekly Architect runs and evaluate whether the memo format is useful.
- Consider adding a shared `agent-core` skill in the future for common primitives.

## Open Architectural Questions
- How should on-demand subagents (like Platform Researcher) be invoked and scoped?
- What isolation level is ideal between agent profiles vs. shared profiles?
