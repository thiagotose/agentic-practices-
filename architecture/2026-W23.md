# Architecture Memo – 2026-W23

## Theme of the Week
Emergence of a "shared agent substrate" pattern. Multiple agents now need common primitives (memory, voice, research discipline, output standards). This is the natural next layer of abstraction.

## Observed Patterns
- Chronicler, Architect, and Closer all repeat similar setup work (folders, indexes, voice rules, cron wiring).
- Platform Researcher was created as a subagent — this suggests a hierarchy: scheduled agents vs on-demand researchers.
- The `agent-core` skill was introduced to reduce duplication.

## Design Proposals

**Proposal: Formalize a two-tier agent model**
- Tier 1: Scheduled narrative agents (Chronicler, Architect, Closer)
- Tier 2: On-demand specialist agents (Platform Researcher and future specialists)
- **Rationale**: Tier 1 agents synthesize and propose. Tier 2 agents answer deep questions when called.
- **Recommended owner**: Thiago + future "Agent Runtime" work

**Proposal: Extract a minimal `agent-runtime` skill**
- Common cron patterns, profile usage, output file naming conventions, and Telegram summary format.
- This would sit alongside `agent-core` (philosophy) and handle mechanical consistency.

## Cross-Repo Observations
- `execute-ai/moura-client-portal` continues to be the production proving ground.
- `personal-workflows` finance tooling is evolving rapidly — potential future "reference implementation" for internal tooling patterns.

## Recommended Experiments
- Run the new `agent-core` skill through Chronicler and Architect for one week and measure duplication reduction.
- Create one more on-demand researcher (e.g., `competitor-researcher`) to test the Tier 2 pattern.

## Open Architectural Questions
- Should Tier 2 agents get their own dedicated profiles, or continue using `chronicler`?
- How do we handle agent-to-agent handoff logging so the Chronicler can capture cross-agent activity?
