- **The Moura inbox is also Execute AI's first reusable product.** "Inbox where AI drafts, human approves" generalizes to every small service business — lawyers, accountants, agencies, coaches. So Moura ships, and the product asset accrues.
- **Hybrid bootstrap strategy locked in:** ship Moura (pro bono, wife's business = safe to iterate) while building the Execute AI context layer in parallel. No revenue pressure yet = rare luxury to design the operating model before market pressure compromises it.

## Reusable insights
- **`CLAUDE.md` per solution is a Real Move.** The Moura one made Hermes effective in 30 seconds. Same discipline goes for `lead-follow-up` and every future solution. Rename pattern: agents read `AGENTS.md` automatically — consider symlinking or migrating naming so any agent (not just Claude Code) gets the context for free.
- **Skill-driven agents > prompt-driven agents.** The Chronicler's behavior lives in a versioned markdown skill, not in a one-shot prompt. Same pattern applies to every future Execute AI agent. The skill IS the agent's spec.
- **Profile-per-agent for isolation.** Each Execute AI agent gets its own Hermes profile = own memory, own voice, own skills loadout. Stops the dev profile from being polluted with cron-agent context.
- **"Agents propose, humans dispose" — for now.** Trust ladder. No agent writes to live systems unsupervised. Bar lowers as patterns prove out.
